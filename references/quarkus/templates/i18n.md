# Qute Internationalization (i18n)

## Overview

Qute provides type-safe message bundles for internationalization. Messages are defined
as Java interfaces and resolved at build time. Localized variants are stored in
`.properties` files alongside the interface.

## Message Bundle Basics

### Define the bundle interface

```java
import io.quarkus.qute.i18n.Message;
import io.quarkus.qute.i18n.MessageBundle;

@MessageBundle
public interface AppMessages {

    @Message("Hello, {name}!")
    String hello(String name);

    @Message("You have {count} items in your cart.")
    String cart_count(int count);

    @Message("{item} added to cart.")
    String item_added(String item);

    @Message("Are you sure you want to delete {name}?")
    String confirm_delete(String name);
}
```

### Use in Qute templates

```html
<h1>{msg:hello(user.name)}</h1>
<p>{msg:cart_count(cart.size)}</p>
```

The `msg:` namespace is automatically available in all templates for the default
`@MessageBundle`.

## Localized Message Files

Create `.properties` files alongside the interface. The naming convention is:

```
src/main/resources/messages/
  msg.properties            # Default (fallback) -- OR define defaults via @Message
  msg_de.properties         # German
  msg_fr.properties         # French
  msg_pt_BR.properties      # Brazilian Portuguese
```

The file prefix matches the bundle name. The default bundle name is `msg`.

### Example: German localization

```properties
# src/main/resources/messages/msg_de.properties
hello=Hallo, {name}!
cart_count=Sie haben {count} Artikel im Warenkorb.
item_added={item} wurde zum Warenkorb hinzugefuegt.
confirm_delete=Moechten Sie {name} wirklich loeschen?
```

Message keys correspond to method names in the interface (with `_` mapping).
Files must be UTF-8 encoded.

### Defining defaults in properties vs annotations

You can define default messages in `@Message` annotations OR in `msg.properties`.
If `@Message` has no value, Qute looks for the key in the default properties file:

```java
@MessageBundle
public interface AppMessages {

    @Message  // looks up "hello" in msg.properties
    String hello(String name);

    @Message("Fallback text for {name}")  // annotation is the default
    String greeting(String name);
}
```

## Named Bundles

Use multiple bundles to organize messages by feature:

```java
@MessageBundle("admin")
public interface AdminMessages {

    @Message("User {name} has been deactivated.")
    String user_deactivated(String name);
}
```

```html
{! Use the bundle name as the namespace !}
<p>{admin:user_deactivated(user.name)}</p>
```

Localized files: `admin_de.properties`, `admin_fr.properties`, etc.

## Locale Resolution

Qute resolves the locale for each template render. The resolution order:

1. Explicit locale set on the `TemplateInstance`
2. The CDI request-scoped locale (set by a filter or interceptor)
3. The default locale (`quarkus.default-locale`)

### Set locale explicitly on render

```java
@GET
@Produces(MediaType.TEXT_HTML)
public TemplateInstance page(@HeaderParam("Accept-Language") String acceptLang) {
    Locale locale = Locale.forLanguageTag(
        acceptLang != null ? acceptLang.split(",")[0] : "en");
    return products.data("products", productService.listAll())
        .setAttribute("locale", locale);
}
```

### Resolve locale from Accept-Language header

```java
@Provider
@Priority(Priorities.HEADER_DECORATOR)
public class LocaleFilter implements ContainerRequestFilter {

    @Inject
    io.quarkus.qute.i18n.Localized.Instance localeInstance;

    @Override
    public void filter(ContainerRequestContext ctx) {
        List<Locale> acceptable = ctx.getAcceptableLanguages();
        if (!acceptable.isEmpty()) {
            localeInstance.set(acceptable.get(0));
        }
    }
}
```

### Configuration

```properties
# Default application locale
quarkus.default-locale=en

# Supported locales (for build-time validation)
quarkus.locales=en,de,fr,pt-BR
```

## Inject Localized Bundles in Java

```java
@ApplicationScoped
public class NotificationService {

    @Localized("de")
    AppMessages deMessages;

    @Inject
    AppMessages defaultMessages;  // uses request locale or default

    public String greet(String name, String lang) {
        if ("de".equals(lang)) {
            return deMessages.hello(name);
        }
        return defaultMessages.hello(name);
    }
}
```

## HTMX Language Switching

Let users switch languages without a full page reload:

### Template

```html
{! Language selector -- sends full page re-render with new locale !}
<nav aria-label="Language">
    <a hx-get="/ui/products"
       hx-target="body"
       hx-push-url="true"
       hx-headers='{"Accept-Language": "en"}'
       aria-label="English">EN</a>
    <a hx-get="/ui/products"
       hx-target="body"
       hx-push-url="true"
       hx-headers='{"Accept-Language": "de"}'
       aria-label="Deutsch">DE</a>
    <a hx-get="/ui/products"
       hx-target="body"
       hx-push-url="true"
       hx-headers='{"Accept-Language": "fr"}'
       aria-label="Francais">FR</a>
</nav>
```

### Persist language preference

For session-based locale persistence, store the preference in a cookie:

```java
@Path("/ui/language")
@ApplicationScoped
public class LanguageResource {

    @POST
    @Consumes(MediaType.APPLICATION_FORM_URLENCODED)
    public Response setLanguage(
        @FormParam("lang") String lang,
        @HeaderParam("HX-Current-URL") String currentUrl
    ) {
        NewCookie cookie = new NewCookie.Builder("lang")
            .value(lang)
            .path("/")
            .maxAge(365 * 24 * 60 * 60)
            .build();
        return Response.seeOther(URI.create(currentUrl != null ? currentUrl : "/"))
            .cookie(cookie)
            .header("HX-Refresh", "true")
            .build();
    }
}
```

Update the locale filter to check the cookie:

```java
@Provider
@Priority(Priorities.HEADER_DECORATOR)
public class LocaleFilter implements ContainerRequestFilter {

    @Override
    public void filter(ContainerRequestContext ctx) {
        // Cookie takes precedence over Accept-Language
        Cookie langCookie = ctx.getCookies().get("lang");
        if (langCookie != null) {
            // Set the locale for Qute message resolution
            CDI.current().select(io.quarkus.qute.i18n.Localized.Instance.class)
                .get().set(Locale.forLanguageTag(langCookie.getValue()));
            return;
        }
        List<Locale> acceptable = ctx.getAcceptableLanguages();
        if (!acceptable.isEmpty()) {
            CDI.current().select(io.quarkus.qute.i18n.Localized.Instance.class)
                .get().set(acceptable.get(0));
        }
    }
}
```

## HTML lang Attribute

Always set the `lang` attribute on `<html>` to match the active locale:

```html
{! base.html !}
<!DOCTYPE html>
<html lang="{locale.language}">
<head>
    <meta charset="UTF-8">
    <title>{msg:page_title}</title>
</head>
<body>
    {#insert content /}
</body>
</html>
```

Pass the locale from the endpoint:

```java
return page.data("products", items)
    .data("locale", resolvedLocale);
```

## Testing i18n

```java
@QuarkusTest
class I18nTest {

    @Test
    void defaultLocale_rendersEnglish() {
        given()
            .accept("text/html")
            .header("Accept-Language", "en")
        .when()
            .get("/ui/products")
        .then()
            .statusCode(200)
            .body(containsString("Hello"));
    }

    @Test
    void germanLocale_rendersGerman() {
        given()
            .accept("text/html")
            .header("Accept-Language", "de")
        .when()
            .get("/ui/products")
        .then()
            .statusCode(200)
            .body(containsString("Hallo"));
    }
}
```

### Unit test message bundles directly

```java
@QuarkusTest
class AppMessagesTest {

    @Inject AppMessages messages;
    @Localized("de") AppMessages deMessages;

    @Test
    void hello_english() {
        assertEquals("Hello, Ada!", messages.hello("Ada"));
    }

    @Test
    void hello_german() {
        assertEquals("Hallo, Ada!", deMessages.hello("Ada"));
    }
}
```
